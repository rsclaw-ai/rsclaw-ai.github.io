## 一、内容定位

### 1. 目标受众画像

| 受众 | 特征 | 核心需求 | 内容偏好 |
|------|------|----------|------|
| **独立开发者** | 熟悉 Rust/Go/Python，有 GitHub 使用经验 | 本地 AI 智能体搭建、多模型接入、自动化工作流 | 技术教程、CLI 指南、配置参考 |
| **AI 爱好者** | 关注前沿 AI 技术，喜欢折腾新工具 | 记忆系统、智能体编排、插件扩展 | 功能解读、对比评测、使用技巧 |
| **企业技术团队** | 需要私有化部署，关注安全与合规 | 多通道接入、团队协作、权限管理 | 架构文档、安全白皮书、部署指南 |
| **自媒体/内容创作者** | 需要 AI 辅助内容生产 | 多渠道分发、定时任务、批量处理 | 场景案例、工作流模板、效率提升技巧 |

### 2. 内容调性

- **专业但不晦涩** — 技术深度到位，但面向非 Rust 用户也能读懂
- **实用主义** — 每篇文章必须解决一个具体问题或提供可操作的价值
- **开源友好** — 鼓励社区贡献，文档风格与 GitHub README 一致
- **螃蟹人格** — 偶尔融入产品吉祥物"螃蟹"的趣味元素，但不喧宾夺主

### 3. 差异化优势

RsClaw 在 AI 智能体领域的独特卖点，所有内容需围绕这些差异展开：

- **Rust 原生性能** — 二进制仅 ~17MB，内存占用远低于同类 Node/Python 方案
- **全渠道统一接入** — 13+ 消息通道，一个智能体同时服务多平台
- **本地优先的长期记忆** — redb + hnsw_rs + tantivy 纯本地向量数据库，零数据外传
- **多智能体编排（A2A）** — 跨机器协作，不是玩具级多 agent
- **Tauri 桌面客户端** — 图形界面 + CLI 共存，开箱即用
- **OpenClaw 无缝迁移** — `rsclaw migrate` 一条命令完成迁移
- **真正的开源** — GitHub 开源，社区驱动

- 如何搭建本地 AI 智能体 / AI 智能体记忆系统怎么实现

### 2. 技术 SEO

- **URL 规范** — `/docs/category/slug`，全小写英文，短横线分隔
- **结构化数据** — Schema.org Article、BreadcrumbList、FAQPage
- **性能优化** — Lighthouse ≥ 90，图片 WebP，字体子集化
- **内容 SEO** — H1/H2/H3 层级清晰，首段含关键词，内部链接密度 ≥ 30%
